# Library system

**Описание**: Система учёта библиотеки — это программное обеспечение для автоматизации учёта книг, журналов и читателей в библиотеках. Решает проблему низкой эффективности ручного учета выдачи и возврата литературы, минимизации человеческого фактора и оперативного контроля состояния фонда. Приложение упрощает ведение библиографических данных, отслеживает состояние экземпляров и помогает эффективно управлять процессом выдачи, возврата и начисления штрафов, улучшая работу сотрудников.

  - **Технологический стек**:   Проект реализован на языке программирования Java 17 с использованием фреймворка JavaFX 17 для создания графического пользовательского интерфейса. Сборка осуществляется с помощью Maven. Программное обеспечение представляет собой автономное десктопное приложение (standalone), не требующее запуска в составе сторонних платформ или серверов.
  - **Статус**:   В данный момент проект находится на бета-стадии. Реализованы основные функции; ведётся тестирование и доработка интерфейса. Подробнее об изменениях см. в(CHANGELOG.md).

Назначение пакетов src
• controller  
  Содержит контроллеры, реализующие связь между пользовательским интерфейсом (JavaFX UI) и бизнес-логикой приложения. Отвечает за обработку событий интерфейса и передачу данных между представлением и моделью.
• model  
  Определяет основные бизнес-сущности и логику работы с объектами библиотеки: книги (TBook), журналы (TJournal), читатели (TReaderCard), учёт выдач (TLoan), а также статусы и результаты операций.
• service  
  Содержит сервисы и вспомогательные компоненты для реализации бизнес-логики и управления данными, например, сервисы по работе с библиотекой, хранением и обработкой операций.
• resources  
  В данный каталог помещаются файлы ресурсов приложения — стили (CSS), FXML-шаблоны интерфейса, изображения и т. д.

## Зависимости

Для корректной работы программного обеспечения необходимы следующие компоненты:

1. Java Development Kit (JDK) 17  
  Основной язык программирования: Java 17.  
  Требуется установленная JDK версии 17 или новее.

2. JavaFX 17  
  Фреймворк для графического пользовательского интерфейса.  
  Необходима совместимая версия JavaFX (17+).  
  В случае использования JDK, не включающей JavaFX, необходимо установить JavaFX отдельно (например, с помощью Maven, как зависимость).

3. Maven 3.6+  
  Система сборки и управления зависимостями.  
  Все внешние библиотеки автоматически подтягиваются через pom.xml.

4. Git  
  Для клонирования репозитория или совместной работы над проектом.

5. Поддержка GUI  
  Операционная система и рабочая среда должны поддерживать запуск JavaFX-приложений (Windows, Linux, macOS).



## Установка

Для начала работы с системой учёта библиотеки пользователю необходимо:
1. Склонировать репозиторий с исходным кодом проекта   git clone <https://github.com/Ekaterina01122002/---.git>
2. Открыть проект в любой современной Java IDE (например, IntelliJ IDEA), убедившись в наличии установленного JDK версии 17 или выше.
3. Собрать проект через систему сборки Maven.
4. Запустить приложение с помощью команды: mvn javafx:run
5. После старта приложения откроется графический интерфейс, где доступны вкладки для работы с книгами, журналами, читателями и операциями выдачи/возврата.

## Проверка ПО

Для проверки корректности работы программы необходимо провести следующие шаги и убедиться, что система работает в соответствии с требованиями:
1. Добавление записей: Добавить новую книгу и новый журнал через интерфейс; Проверить, что они появились в списках с корректными атрибутами; Зарегистрировать нового читателя; убедиться, что данные отображаются корректно.
2. Выдача литературы: Осуществить выдачу одной или нескольких единиц литературы зарегистрированному читателю; Убедиться, что статус книги или журнала изменился на "выдана", у читателя увеличилось количество выданных экземпляров, дата возврата назначена автоматически.
3. Проверка лимитов: Попробовать выдать читателю литературы больше, чем установленный лимит; Программа должна не позволить это сделать и показать соответствующее предупреждение.
4. Возврат литературы: Провести возврат ранее выданной книги или журнала; Проверить, что статус литературы снова "в наличии", а у читателя уменьшилось количество на руках; В случае просрочки возврата программа должна корректно начислить штраф и отразить это в карточке читателя/выдаче.
5. Редактирование и удаление: Изменить данные о книге, журнале, читателе; Удалить запись и убедиться, что она исчезла из каталога/списка.
6. Поиск и фильтрация: Использовать поиск по различным критериям (название, автор, ID), убедиться, что фильтрация срабатывает правильно и выдаёт ожидаемые результаты.
