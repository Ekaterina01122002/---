# Объектно-ориентированная модель библиотечной системы
# Описание
Проект представляет собой курсовую работу по дисциплине «Объектно-ориентированное программирование» и посвящён разработке программной системы для учёта книг и читателей библиотеки. Система позволяет вести электронный каталог книг и журналов, регистрировать читателей, отслеживать выдачу и возврат, а также управлять статусом библиотечных единиц и вести учёт штрафных баллов за просрочку возврата.
# Применение
1. Учебные заведения
2. Публичные и муниципальные библиотеки
3. Музеи и культурные учреждения
# Технологический стек проекта
1. Язык программирования - Java 17
2. Графический интерфейс - JavaFX 17+
3. Инструменты и ПО для разработки: IntelliJ IDEA; Git; Maven
4. Автоматизация и сборка - pom.xml  
# Как начать работу с проектом?
Для начала работы с системой учёта библиотеки пользователю необходимо:
1. Склонировать репозиторий с исходным кодом проекта   git clone <https://github.com/Ekaterina01122002/---.git>
2. Открыть проект в любой современной Java IDE (например, IntelliJ IDEA), убедившись в наличии установленного JDK версии 17 или выше.
3. Собрать проект через систему сборки Maven.
4. Запустить приложение с помощью команды: mvn javafx:run
5. После старта приложения откроется графический интерфейс, где доступны вкладки для работы с книгами, журналами, читателями и операциями выдачи/возврата.
6. # Проверка ПО
7. Для проверки корректности работы программы необходимо провести следующие шаги и убедиться, что система работает в соответствии с требованиями:
1. Добавление записей
   • Добавить новую книгу и новый журнал через интерфейс.
   • Проверить, что они появились в списках с корректными атрибутами.
   • Зарегистрировать нового читателя; убедиться, что данные отображаются корректно.
2. Выдача литературы
   • Осуществить выдачу одной или нескольких единиц литературы зарегистрированному читателю.
   • Убедиться, что статус книги или журнала изменился на "выдана", у читателя увеличилось количество выданных экземпляров, дата возврата назначена автоматически.
3. Проверка лимитов
   • Попробовать выдать читателю литературы больше, чем установленный лимит.
   • Программа должна не позволить это сделать и показать соответствующее предупреждение.
4. Возврат литературы
   • Провести возврат ранее выданной книги или журнала.
   • Проверить, что статус литературы снова "в наличии", а у читателя уменьшилось количество на руках.
   • В случае просрочки возврата программа должна корректно начислить штраф и отразить это в карточке читателя/выдаче.
5. Редактирование и удаление
   • Изменить данные о книге, журнале, читателе.
   • Удалить запись и убедиться, что она исчезла из каталога/списка.
6. Поиск и фильтрация
   • Использовать поиск по различным критериям (название, автор, ID), убедиться, что фильтрация срабатывает правильно и выдаёт ожидаемые результаты.
